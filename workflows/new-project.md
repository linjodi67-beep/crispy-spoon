# New Project Workflow

Use this when starting a new Codex-assisted project.

## Goal

Turn a rough idea into a small working project with a clean GitHub history.

## Steps

1. Name the project.
2. Decide the first useful version.
3. Create the project under `C:\Users\LINJO\CodexProjects`.
4. Add a README with purpose, setup, and usage.
5. Build the smallest working version first.
6. Verify it runs locally.
7. Save the first version to GitHub.
8. Track follow-up ideas as issues.

## First Version Rule

The first version should be small enough to finish and test in one focused session.

Good first versions:

- a usable page instead of a full platform
- one automation instead of a whole operations system
- one report template instead of a full analytics suite
- one game loop instead of a finished game
