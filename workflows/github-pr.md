# GitHub Pull Request Workflow

Use this when a project change is ready to review and merge.

## Goal

Make each finished change easy to understand, test, and preserve.

## Steps

1. Check what changed.
2. Run the relevant verification.
3. Write a clear summary.
4. Create a pull request.
5. Review files and comments.
6. Fix review feedback if needed.
7. Merge when the change is ready.

## Good Pull Request Size

A good pull request should answer:

- What changed?
- Why did it change?
- How was it checked?
- What still needs follow-up?

## Codex Prompt

```text
Review my current changes, run the right checks, write a clear PR summary, and prepare a GitHub pull request.
```
